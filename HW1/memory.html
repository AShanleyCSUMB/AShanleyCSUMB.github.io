Virtual Memory

Paging, page tables, and page faults

The Big Idea

Virtual memory lets each process act like it has a large, private address space even when physical RAM is limited. The OS and hardware translate virtual addresses into physical addresses using data structures such as page tables. This makes multitasking easier and helps isolate programs from each other.

Paging

Paging divides memory into fixed-size chunks. Virtual memory is split into pages, and physical RAM is split into frames. A page table tells the hardware where each virtual page currently lives in physical memory.

Page Faults

If a process tries to use a page that is not currently in RAM, the CPU triggers a page fault. The OS then loads that page from storage (like an SSD swap area or backing file) into a free frame, updates the page table, and restarts the instruction.

Why It Helps

  • Lets programs use more memory than RAM by moving inactive pages to storage.
  • Improves isolation: each process can have a separate address space.
  • Enables sharing: some pages (like shared libraries) can be mapped into multiple processes.
