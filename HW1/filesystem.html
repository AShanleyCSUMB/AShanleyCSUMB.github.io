What a File System Provides

A file system is the OS component that turns raw storage blocks into files and folders. It defines how data is named, stored, located, and protected. It also tracks metadata like permissions, timestamps, and file size.

Metadata & Inodes (Unix-like systems)

In many Unix-like file systems, a file’s metadata is stored in a structure called an inode. The inode keeps information such as file size, ownership, permissions, and pointers to the data blocks, while the filename lives in a directory entry that references the inode.

Directories

A directory is like a table that maps names to file identifiers (like inode numbers). This is why the same underlying file data can sometimes be referenced by multiple names (hard links), depending on the file system.

Practical Goals

  • Reliability: reduce corruption risk (journaling, checksums, etc.).
  • Performance: organize blocks to reduce slow random disk access.
  • Security: permissions and access controls help prevent unauthorized reads/writes.
